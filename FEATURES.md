## Design System

All pages share the following token values:

```typescript
const BG     = "#07070F";  // Page background (near-black)
const CARD   = "#12121F";  // Card background
const BORDER = "#1E1E32";  // Border color
const GREEN  = "#00FF88";  // Primary accent (lime green)
const PURPLE = "#6B4FBB";  // Secondary accent
const TEXT   = "#F2F2F8";  // Primary text
const MUTED  = "#9898B3";  // Secondary / muted text
```

### UI Patterns in Use

1. **Filter buttons + card grid + sticky side detail panel** — used for collections (saints, martyrs, masterworks, revivals, books, etc.)
2. **Accordion expand/collapse** — used for theology points, FAQ, multi-point explanations
3. **Tabs with typed literal union** — `type Tab = "tab1" | "tab2" | "tab3"` with `useState<Tab>("tab1")`
4. **Vertical timeline** — used for biblical storyline, church history, phases
5. **Stats banner** — 3-4 key numbers at top of resource pages
6. **Two-column comparative grid** — used for political issues, atonement strengths/weaknesses, etc.

## Build & Deployment Notes

- **Branch**: `claude/vibrant-pascal-eOo9I`
- **Build command**: `npm run build`
- **All pages are static** (`○` in Next.js build output)
- **No TypeScript errors** on any committed page
- **Common TypeScript traps** to avoid:
  - Use `undefined` not `null` or `None` for optional fields
  - Always close string literals inside object properties (e.g., `color: "#10B981"` not `color: "#10B981,`)
  - Tab type must be a literal union: `type Tab = "a" | "b"` not `string`
